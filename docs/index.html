Browser Support

Flex was designed to work with the most major browsers. If a module has issues in a specific browser it's mentioned in the Browser Support section of that specific module.

How to use

After you download and unzip Flex you will be left with 2 folders:

  1. Docs: The documentation website.
  2. Src: Contains A basic HTML template to quickly get you started, images, flex.min.css and flex.min.js

If you would like a stripped down version of Flex - maybe with fewer modules - you can download the entire framework from github and compile it yourself. The following tools are needed:

  • SASS 1 To compile the CSS Files
  • Jammit To compile the Javascript Files

Flexible Grid

The flexible grid is based on the concepts advocated by Ethan Marcotte in his book Responsive Webdesign. The formula behind the grid math calculations is fairly simple: target % context = result.

Everything you need to do in order to use the grid is to wrap the columns in .l-row parent containers. Basically the rows keep the columns constrained at specific dimensions that change based on the user's screen size.

Caveats

The entire grid uses percentages to layout the structure. Following this convention during the development process is vital in order to preserve the integrity of the flexible grid. With that in mind here are a few tips:

  • When calculating the margins the context is the width of the element's container.
  • When calculating the paddings the context is the width of the element itself.
From pixels to percentages

Let's say you have a <div class="contactBox"></div> somewhere on the page and it's values are currently expressed in pixels:

  • width: 300px; padding: 30px; margin: 25px;

Knowing that the parent's width is 780px let's convert those pixel based values into percentages by making use of our formula.

  • Width: 300 / 780 = .384615385. Moving the decimal over two places we're left with 38.4615385%.
  • Padding: 30 / 300 = 10%.
  • Margin: 25 / 780 = 3.205128205%.

Block Grid

Used for displaying a horizontal list of items without needing extra mark-up like rows and columns. It basically behaves like inline-block.

Note: Be aware that while apparently it scales like the flexible grid system it does not convert itself to block-level styling at smaller view ports.

Class

  • .l-blockX whereas X represents the amount of items per row.

Usage

Place the required class on a parent element and it's direct child elements will behave appropiately. Easy as that!

Forms

The form module comes in two formats:

  1. Horizontally Aligned
  2. Vertically Aligned

To use either one of these, a class on the parent <form> element is required. If you would like your html to follow a different structure but still have the beautifully form inputs just wrap the element in a container (usually a div) and set it's class to .frm--input.

The input, select & textarea elements have the box-sizing set to border-box allowing us to have fluid form fields and take advantage of the Flexible Grid without having to worry about the applied padding and border.

Forms & the Flexible Grid

The form module and the grid system go hand in hand. Simply apply one of the grid classes to the form fields.

Class

  • .frm (horizontally aligned) 1

1 Horizontal forms will automatically adjust to become vertically structured according to the screen view size.

Extend

  • .frm-vertical must extend .frm

Tooltip Requires Javascript

Using the tooltip module is pretty simple:

  1. Make sure the jquery.tooltip.js plugin is included.
  2. Apply data-trigger="tooltip" on any element element you want the tooltip to be activated on.
  3. Use data-content attribute to supply content to the tooltip. If the element is an anchor you can just use the title attribute.
Performance Improvements

By default the tooltip plugin is not initialized until the very last moment (i.e. users hovers the html element). The technique is called deferred initialization and it's accomplished by using event capturing. Unfortunately it only works on browsers that support the W3C event model.

Bubble Requires Javascript

Using the bubble module is pretty simple:

  1. Make sure that jquery.tooltip.js and jquery.bubble.js plugins are included.
  2. Apply data-trigger="bubble" on any element element you want the bubble to be activated on.
  3. Use data-content="dummy text" to supply content to the bubble. If the element is an anchor you can simply use the title attribute.
Performance Improvements

By default the bubble plugin is not initialized until the very last moment (i.e. users triggers the mouseenter/focus event on the html element). The technique is called deferred initialization and it's accomplished by using event capturing. Unfortunately it only works on browsers that support the W3C event model.
